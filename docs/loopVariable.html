Programmiertechniken - Variablen und Schleifen

Variablen

In der Informatik sind Variablen eine Art Platzhalter oder Container für Daten und Informationen. Der Inhalt einer Variablen kann sich kann sich jederzeit ändern. Um eine Variablen nutzen zu können, muss im Programmcode für jede Variable der Variablenname und der Datentyp (z.B. Text, Zahl, Buchstabe, ...) festgelegt werden. Diese Festlegung der Eigenschaften einer Variablen wird Deklaration oder Variablendeklaration genannt (von declare lat. bekannt machen, bezeichnen).

Variablenname

Der Variablenname soll immer beschreiben, welche Funktion oder welchen Inhalt eine Variable hat. Es ist sinnvoll, für Variablen Substantive wie messWertSensorLinks oder pinRoteLED zu verwenden. Variablennamen sollten immer mit einem Kleinbuchstaben beginnen. Die Nutzung eines Unterstrichs zum Trennen einzelner Wörter soll vermieden werden. Stattdessen kann zu verbesserten Lesbarkeit bei zusammengesetzten Wörtern die Großschreibung verwendet werden. Statt zustand_motor_links sollte die Variable besser zustandMotorLinks genannt werden.

Datentyp

Variablen können für viele verschiedene Datentypen festgelegt werden. Der Datentyp gibt an, ob die Variable z.B. Zahlen oder Buchstaben beinhalten kann.

Die wichtigsten Datentypen beim Arduino:

  • bool Speichert nur die Zustände bzw. Werte true (1) oder false (0), also wahr oder falsch ab.
  • int Abhängig vom Arduinoboard werden ganzzahlige Werte zwischen -32.768 und 32.767 (16 bit Wert beim UNO, Leonardo) oder zwischen -2.147.483648 und 2.147.483.647 (32 bit Werte beim MKR1000 and Zero) gespeichert.
  • unsigned int Ähnlich wie int, allerdings werden nur positive ganzzahlige Werte gespeichert. Der Wertebereich ist 0 bis 65.535 = 2^16 - 1.
  • long Speichert ganzzahlige Werte zwischen -2.147.483648 und 2.147.483.647.
  • unsigned long Ähnlich wie long, allerdings werden nur positive ganzzahlige Werte gespeichert. Der Wertebereich ist 0 bis 4.294.967.295 = 2^32 - 1.
  • float Kann 4 bit Fließkommazahlen (Dezimalbrüche mit festgelegter Genauigkeit) im Bereich -3,4028235E+38 bis 3,4028235E+38 speichern.

Oft werden auch alternative Datentypen benutzt, die die Kompatibilität mit verschiedenen Mikrokontrollern verbessern soll:

  • uint8_t , uint16_t, uint32_t, uint64_t Speichern positive ganzzahlige Werte im mit der angegeben Bitbreite (8, 16, 32, 64 bit)
  • int8_t , int16_t, int32_t, int64_t Speichert positive und negative ganzzahlige Werte im mit der angegeben Bitbreite (8, 16, 32, 64 bit)

Beispiele Deklaration

int pinRoteLed;   // Die Variable mit dem Namen pinRoteLed kann nur ganzzahlige Werte annehmen
float fehlerWert; // Die Variable mit dem Namen fehlerWert kann auch eine Fließkommazahl (Dezimalbruch) sein.

Zuweisung

Nachdem eine Variable deklariert wurde, ist es möglich ihr auch einen Wert zuzuweisen. Zuweisungen werden mit dem = Operator durchgeführt. Auch wenn es nicht immer ganz offensichtlich ist, sollte der = Operator nicht mit dem Gleichheitszeichen (=) in der Mathematik verwechselt werden. Das Gleichheitszeichen drückt aus, dass etwas gleich ist. Der Zuweisunges-Operater weist der Variablen einen Wert zu. Dieser Unterschied wird später wichtig, wenn Vergleichsoperatoren wie zum Beispiel == verwendet werden.

Zuweisung können getrennt von der Variablendeklaration durch geführt werde. Wird einer Variablen das erste Mal ein Wert zugewiesen, spricht man auch von der Initialisierung der Variablen. Bevor eine Variable initialisiert wurde, ist der Wert der Variablen unbekannt.

int auszeitRoteLed;    // Die Variable mit dem Namen auszeitRoteLed kann nur ganzzahlige Werte annehmen
auszeitRoteLed = 1000; // Zuweisung und Initialisierung
/*
 *  beliebiger Code ...
 */
auszeitRoteLed = 300;  // eine erneute Zuweisung überschreibt die Variable mit einem neuen Wert

Da jede Variable deklariert und auch initialisiert werden muss, kann das auch in einer Programmzeile gemacht werden.

float fehlerWert = 3.1415; // Deklaration und erst Zuweisung, also Initialisierung in einer Zeile.

Die for Schleife

Die for-Schleife gehört zu den Kontrollstrukturen eines Programms und erlaubt das wiederholende Ausführen eines Programmabschnitts.
