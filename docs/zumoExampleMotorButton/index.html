Getting Started - Motoren und Taster

Nachdem der Treiber und die Bibliothek installiert worden sind, kann der Zumo programmiert werden. Da der Zumo auch mit Arduino programmiert wird, ist der Einstieg nicht besonders schwer. Allerdings gibt es einige neue Befehle, welche zur Steuerung des Roboters notwendig sind. In der vereinfachten Befehlsreferenz finden Sie ein Übersicht der wichtigsten Befehle. Schauen Sie sich die Referenz immer mal wieder an, um sich mit den Programmiermöglichkeiten des Zumos vertraut zu machen.

Schnelleinstieg

Für einen schnellen Einstig und als guten Überblick über die Grundfunktionen lernen Sie im folgenden Beispiel Methoden zum Steuern der

  • Motoren
  • LEDs und
  • Taster

kennen. Nach diesem Schnelleinstieg sollten Sie mithilfe der Befehlsreferenz in der Lage sein auch Sensoren (Bodensensor, Hindernisssensor) in Ihr Programm einzubinden. Erläuterungen finden Sie als Kommentar im Code oder etwas ausführlicher weiter unten.

GettingStarted.ino
/*
   Einfaches Beispiel als Schnelleinstieg in die Programmierung des Zumo. Nach dem Starten
   wird der Roboter verschieden Aktionen ausführen. Die Auswahl der Aktion geschieht
   durch das Drücken einer der drei Tasten.
*/

#include <Zumo32U4.h>    // Die Bibliothek sollte immer als erstes eingebunden werden

// Für fast jedes Bauteil (Taster, Motoren, Display, Sensoren) wird in der Zumo-Bibliothek
// eine eigene Klasse mit Methoden zum Konfigurieren, Steuern und Auslesen zur Verfügung
// gestellt.
Zumo32U4LCD lcd;          // Display (8x2)
Zumo32U4ButtonA buttonA;  // Taster A
Zumo32U4ButtonB buttonB;  // Taster B
Zumo32U4ButtonC buttonC;  // Taster C
Zumo32U4Motors motors;    // Motoren


// Die Motorgeschwindigkeit kann Werte zwischen -400 ... 400 annehmen
int leftSpeed = 200;
int rightSpeed = 200;

void setup() {
  lcd.clear();             // Display löschen
  lcd.print(F("Press A")); // Textausgabe
  buttonA.waitForButton(); // Programm pausiert, bis Taster A gedrückt wurde
  lcd.clear();
  lcd.print(F("A  B  C"));
}

void loop() {

  // 2 Sekunden geradeaus fahren.
  if ( buttonA.isPressed() ) {   // Abfrage, ob Taster gedrückt wurde
    ledYellow(1);                // Gelb LED einschalten
    delay(500);                  // 0,5 Sekunden warten
    motors.setSpeeds(leftSpeed, rightSpeed); // Motorgeschwindigkeit setzen
    delay(2000);
    motors.setSpeeds(0, 0);
    ledYellow(0);                // Gelbe LED ausschalten
  }

  // Langsames Beschleunigen und Abbremsen.
  if (buttonB.isPressed()) {
    ledGreen(1);
    delay(500);
    // Beschleunigen
    for (int speed = 0; speed < 401; speed++)
    {
      motors.setSpeeds(speed, speed);
      delay(6);
    }
    // Abbremsen
    for (int speed = 400; speed > 0; speed--)
    {
      motors.setSpeeds(speed, speed);
      delay(6);
    }
    ledGreen(0);
  }

  // Ein Viereck fahren.
  if (buttonC.isPressed()) {
    ledRed(1);
    delay(500);
    /*
      Eigenen Code einfügen ...
    */
    ledRed(0);
  }
}

LEDs

Die LEDs benötigen keine speziellen Objekte zur Ansteuerung. Für jede LED steht ein eigene Funktion ledRed(), ledYellow, ledGreen zur Verfügung, welche als Argument 0 (aus) oder 1 (an) akzeptieren.

ledYellow(1);  // schaltet die gelbe LED an
ledYellow(0);  // schaltet die gelbe LED aus

Taster

Für die Nutzung der Taster werden drei Klassen (Zumo32U4ButtonA, Zumo32U4ButtonB und Zumo32U4ButtonC) bereitgestellt. Nach dem Erzeugen der entsprechenden Objekte

Zumo32U4ButtonA buttonA;  // Taster A

stehen verschieden Methoden zum Auslesen der Taster bereit. Mit der Zeile

buttonA.waitForButton(); // Programm pausiert, bis Taster A gedrückt wurde

wird die Programmausführung solange pausiert, bis der Taster A gedrückt wurde.

Die Methode isPressed() gibt true zurück, wenn der Taster gedrückt ist, ansonsten false. Diese Methode ist nicht entprellt. Die Methode eignet sich besonders um in Anbhängigkeit der Taster Aktionen ausführen zu lassen. Hierfür wird die Methode mit eine if-else Verzweigung kombibiert.

if ( buttonA.isPressed() ){
  // tue etwas ...
}

Motoren

Für die Motoren steht die Klasse Zumo32U4Motors zur Verfügung. Die wichtigste Methode zum Steuern der beiden Motoren ist setSpeeds(int left, int right). Die Argument geben jeweils die Drehgeschwindigkeit der beiden Motoren an. Die erlaubten Werte liegen im Bereich von -400 bis 400.

motors.setSpeeds(-300, 300); // auf der Stelle rotieren

LCD Display

Aufgaben

  1. Erweitern Sie das obige Programm so, dass Ihr Roboter ein möglichst exaktes Viereck fährt.
  2. Schreiben Sei eine Funktion void drehen(in grad), welche den Roboter um die angegebene Gradzahl drehen lässt. Beachten Sie, dass ein negativer Winkel eine Drehung nach rechts verursacht.
