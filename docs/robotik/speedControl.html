Ein Regler für die Geschwindigkeit

Mit den Motor-Encodern kann der Roboter nicht nur geradeaus fahren oder eine festgelegte Distanz zurücklegen, sondern es ist damit auch möglich, die Geschwindigkeit des Roboters zu regeln. Ohne Geschwindigkeitsregelung würde zum Beispiel ein fester PWM-Wert für den Motor bei abnehmender Batteriespannung zu einem langsam fahrenden Roboter führen.

Vorbetrachtungen zum P-Regler​

Aber wie würde ein Regler für die Geschwindigkeit aussehen? Vom Linienfolger ist bereits der Proportional-Regler bekannt. Angenommen, der Roboter soll mit einer Gechwindigkeit von 2000 Encoder-Impulsen pro Sekunde fahren. Dann würde sich die Regelabweichung (also der Fehler) wie folgt berechnen:

fehler = geschwindigkeit - 2000       // Geschwindigkeit in Impulse pro Sekunde

Mit den Proportionalregler würde dann das Motorsignal für die setSpeeds()-Funktion bestimmt werden:

motorSignal = kp * fehler             // P-Regler für das Motorsignal
setSpeeds(motorSignal,motorSignal)    // einstellen der Geschwindigkeit

Betrachten wir an dieser Stelle die zwei Fälle Geschwindigkeit ist 0 und Geschwindigkeit ist 2000.

Geschwindigkeit ist 0: In diesem Fall ergibt sich ein Fehler von 2000. Mit einem kp=0.1k_p = 0.1kp​=0.1 würde sich ein Motorsignal von 200 ergeben und der Roboter würde anfangen loszufahren.

Geschwindigkeit ist 2000: In dem Fall, wenn der Roboter seine Sollgeschwindigkeit von 2000 erreicht hätte, würde der Fehler 0 sein und sich ein Motorsignal von ebenfalls 0 einstellen, der Roboter würde also wieder stehen bleiben.

Ein Proportional-Regler, der das Motorsignal einfach nur proportional zum Fehler bestimmt, ist also zur Regelung der Geschwindigkeit ungeeignet. Ebenfalls ist der Differential-Regler ungeeignet, da er nur auf Änderungen des Fehlers eingeht. Steht der Roboter einfach nur still, so ändert sich der Fehler nicht und es würde sich für das Motorsignal nur der Wert 0 ergeben.

Integral-Regler​

Der Integralanteil (I-Anteil) eines Regler versucht bleibende Regelabweichungen (steady state error) zu verringern.

Der Integralanteil gibt dem Regler ein Gedächtnis, wodurch Abweichungen von der Sollgröße, die über einen langen Zeitraum bestehen, besonders berücksichtigt werden.

Aus mathematischer Sicht entspricht dieses Gedächtnis dem Aufsummieren der Fehler oder auch dem Integral der Fehler über die Zeit.

∫0te(t) dt≈e(t0) ⋅Δt+e(t1) ⋅Δt+⋯=(e(t0)+e(t1)+⋯ ) Δt\int_{0}^{t} e(t) \text{ d}t \approx e(t_0)\ \cdot \Delta t + e(t_1)\ \cdot \Delta t + \cdots = ( e(t_0) + e(t_1) + \cdots )\ \Delta t∫0t​e(t) dt≈e(t0​) ⋅Δt+e(t1​) ⋅Δt+⋯=(e(t0​)+e(t1​)+⋯) Δt

Für den Pseudocode kann das Integral mit Hilfe der einzelnen Summen \( e(t_i)\cdot \Delta t \) berechnet werden. Zu jedem neuen Zeitpunkt wird zu dem bereits berechneten Integral-Wert der neue Fehler \( e(t_i)\cdot \Delta t \) hinzuaddiert. Dadurch wird aus dem komplizierten Integral eine sehr einfache Addition:

integral = integral + error * dTime // Aufsummieren des Fehler zum Integralanteil

Die, ähnlich wie beim Differential-Anteil, zu einer einfachen Summe wird, wenn bei konstanten Zeitintervallen dTime weggelassen wird:

integral = integral + error // Aufaddieren des Fehlers liefert den Integralanteil

Zur Regelung der Motorgeschwindigkeit wird der Integral-Anteil, ähnlich wie der Proportional-Anteil, mit einem Faktor \(k_i\) gewichtet. Für die Geschwindigkeitsänderung deltaSpeed ergibt sich zusammen mit dem P-Anteil folgender Pseudocode:

integral   = integral + error             // Aufsummieren des Fehler zum Integralanteil
deltaSpeed = error * kp + integral * ki   // PI-Regler mit den Faktoren kp und ki

Integralanteil beschränken​

Wird der Integralanteil nicht unendlich groß, wenn der Fehler kontinuierlich aufaddiert wird?

Im Prinzip sollte der Integral-Anteil nicht unendlich groß werden, da der Fehler sowohl positiv, als auch negativ sein kann und sich beim Aufaddieren somit ausgleichen würde. In der Praxis sieht es jedoch oft anders aus und der Integralanteil kann unerwünscht große Werte annehmen.

Besteht für eine lange Zeit ein positiver Fehler (zum Beispiel durch eine enge Kurve) so ergibt sich ein recht großer Integralanteil. Wird durch den Regler der Fehler nach einer Weile wieder auf Null oder ein wenig ins Negative gebracht, wäre das Ziel erreicht, und der Roboter befände sich auf der Linie. Das Problem ist jedoch, dass der bereits stark angewachsene positive Integralanteil durch das Aufaddieren von wenigen negativen Fehlerwerten nicht schnell genug auf Null zurückgeht.

Bei der Berechnung von deltaSpeed würde der Integralanteil also immer noch den aufaddierten positiven Fehler korrigieren wollen, während der Roboter bereits wieder über der Linie ist. Daher macht es Sinn, für den Integral-Anteil nur eine bestimmte Anzahl von Fehlerwerten zu berücksichtigen.

Wird nur der aktuelle Fehler (ene_nen​) und der vorhergehenden Fehler (en−1e_{n-1}en−1​) berücksichtigt, so würde sich folgender Pseudocode ergeben:

integral = lastError + error;

Sollten mehr als nur die letzte beiden Fehlerwerte berücksichtigt werden, so kann durch den Arduinobefehl constrain() der Wert des Integralanteils auf sinnvolle Werte begrenzt werden:

integral = constrain(integral,-4000,4000);

Vollständiger PID-Regler​

Führt man den P, I und D-Anteil zusammen, erhält man den so genannten PID-Regler. Der PID-Regler ist in der Regelungstechnik sehr weit verbreitet und findet überall dort eine Anwendung, wo ein vorgegebener Wert (Temperatur, Wasserstand oder Linienposition) trotz Abweichungen durch äußere Einflüsse eingehalten werden soll.

Beim PID-Regler ergibt sich die Geschwindigkeitsänderung aus der Summer des Proportional-, des Intergral- und des Differential-Anteils:

deltaSpeed   = (error * kp) + (integral * ki) + (differential * kd)

Pseudocode PID-Regler​

Der gesamte Pseudocode ohne Berücksichtigung von dTime würde dann folgendermaßen aussehen:


wertLinie = 500                             // Sollwert (Sensorwert genau über der Linie)
kp        = 1                               // Proportional-Faktor
ki        = 0.1                             // Integral-Faktor
kd        = 0.1                             // Differential-Faktor
integral  = 0                               // Startwert für Integral-Anteil
error_alt = 0                               // alter Fehlerwert


wiederhole immer
    wertSensor   = leseSensor()             // `wertSensor` sei ein Wert zwischen  0 und 1000
    error        = wertLinie - wertSensor   // Fehlerwert, Regeldifferenz
    integral     = integral + error         // Integral-Anteil 
    differential = error - error_alt
    error_alt    = error
                                     
    deltaSpeed   = error * kp +  integral * ki + differential * kd  // PID-Regler
                                
    leftSpeed    = leftSpeed - deltaSpeed   // Drehzahl links bestimmen
    rightSpeed   = rightSpeed + deltaSpeed  // Drehzahl rechts bestimmen

    begrenze(leftSpeed)                     // Drehzahl auf sinnvolle Werte begrenzen
    begrenze(rightSpeed)                    // (beim Zumo zwischen -400 und +400)
    setMotors(leftSpeed,rightSpeed)         // Aktualisieren der Motordrehzahlen
