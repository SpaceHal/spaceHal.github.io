Funktionen

Eine Funktionen ist ein zusammengefasster Codeblock (Unterprogramm), welcher an beliebiger Stelle eines Programms aufgerufen werden kann. Wird eine Funktion in einem Programm aufgerufen, so wird zuerst der Code der Funktion ausgeführt und anschließend das Programm an der Steile des Funktionsaufrufes weitergeführt. Funktionen können Parameter als Eingabegrößen akzeptieren und berechnet Werte zurückgeben.

Eine Funktion zum Anschalten einer LED

Ein sehr einfaches Beispiel für eine Funktion könnte das Anschalten einer LED sein. Diese Funktion soll keine Eingabeparameter und keine Rückgabewerte haben und beim Aufruf eine vorher festgelegte LED anschalten.

Die Eigenschaften der Funktion werden in der Funktionsdefinition beschrieben. Dort müssen der Name der Funktion (anschalten) und die Datentypen des Rückgabewertes und der Eingabeparameter definiert werden. Außerdem wird wird hier auch der, von der Funktion auszuführende Code angegeben.

Der Datentyp des Rückgabewertes wird bei der Definition vor den Funktionsnamen geschrieben. Wird von der Funktion kein Wert zurückgegeben, wird das Schlüsselwort void genutzt. Werden der Funktion keine Parameter übergeben, so folgen nach dem Funktionsnamen zwei leere runde Klammern.

Der von der Funktion auszuführende Code wird anschließend zwischen zwei geschweifte Klammern {...} geschrieben.

//  Funktionsdefinition für `anschalten()`
void anschaltenLed()               
{
    digitalWrite(PIN_LED,HIGH);  // Eine Funktion kann natürlich auch mehr als nur eine Zeile Code enthalten.
}

Nachdem eine Funktion definiert wurde, kann sie mit ihrem Namen aufgerufen werden:

anschaltenLed();  //  Funktionsaufruf

Beim Aufruf einer Funktion dürfen die runden Klammern nach dem Funktionsnamen nicht fehlen.

Das ganze Programm

Wird noch eine weiter Funktion ausschalten() definiert, so ergibt sich ein sehr verständlicher Code zum Blinken eine LED:

#define PIN_LED 2   

// Eine Funktionsdefinition 
void anschaltenLed(){
    digitalWrite(PIN_LED,HIGH);  
}

// Noch eine Funktionsdefinition 
void ausschaltenLed(){
    digitalWrite(PIN_LED,LOW);   
}

void setup() {
  pinMode(PIN_LED,OUTPUT); 
}

// Sehr verständlicher Code 
void loop() {
  anschaltenLed();  // Aufruf der Funktion 
  delay(1000);   
  ausschaltenLed(); // Aufruf der Funktion                       
  delay(1000);                 
}

Konventionen

Eine wesentliche Aufgabe von Funktionen ist es, ein Programm lesbar zu machen. Aus diesem Grund sind die gewählten Funktionsnamen äußerst wichtig. Folgenden Regeln sollten sie für ihre Funktionsnamen einhalten:

  • Ein Funktionsname beginnt immer mit einem Verb im Infinitiv. Anschließend können weiter Wörter zum Erläutern folgen (schatenLed(), messenSensor(), lesenZustand(), ...)
  • Der Funktionsname beschreibt nur die von der Funktion ausgeführte Aktion.
  • Oft gilt: Je kürzer eine Funktion, desto besser. Schreiben Sie lieber zwei oder drei kurze Funktionen, als eine lange.

Aufgaben

  1. Schreiben Sie die Funktion gruessen() (verwenden Sie keine Umlaute), welche auf dem seriellen Monitor folgenden Satz schreibt:

    Hallo Welt, wie geht es?

    Nutzen Sie ansschließend die Funktion, um den Satz dreimal auszugeben:

    Hallo Welt, wie geht es?
Hallo Welt, wie geht es?
Hallo Welt, wie geht es?

  2. Schreiben Sie eine Funktion aufblinken(), welche ein LED zuerst anschaltet und 0,2 Sekunden später wieder ausschaltet. Nutzen Sie die Funktion, um eine LED blinken zu lassen:

    void loop(){
    aufblinken();
    delay(1000);
}

Funktionen mit Parametern

Sollte die obige Funktion anschalten() auch zum Anschalten einer anderen LED genutzt werden, müsste der Pin für die LED geändert werden. Dass kann entweder direkt im Code der Funktion oder mit einem Funktionsparameter gemacht werden.

void schaltenLed(bool pZustand){
  digitalWrite(LED, pZustand);
}

Funktionen mit Rückgabewert

Mit Rückgabewert, mit Parameter

bool umschaltenLed(int pPinLed){
  static bool state = false;   // statische Variable, wird nach dem Beenden der Funktion nicht gelöscht 
  state != state;
  digitalWrite(pPinLed, state);
  return state;
}
